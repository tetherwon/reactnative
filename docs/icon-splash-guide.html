1. 먼저 알아둘 점

  • 아이콘·스플래시는 설정 파일 app.json 이 가리키는 이미지로 결정됩니다. 이미지를 같은 경로·같은 파일명으로 교체하면 가장 간단합니다.
  • 이 화면들은 실제 빌드(dev build / production) 에서만 보입니다. Expo Go 에서는 Expo 기본 아이콘이 보이니 빌드해서 확인하세요.
  • 모든 이미지는 PNG 권장. (스플래시·안드로이드 전경은 투명 배경 가능 / iOS 메인 아이콘은 투명 불가)
가장 빠른 방법. 아래 표의 경로에 있는 파일을 동일한 이름·동일한(또는 더 큰) 크기의 새 이미지로 덮어쓰기 → 6번의 반영 명령 실행. 끝.

3. 앱 아이콘

3-1. 공통 메인 아이콘 — icon.png

assets/images/icon.png 를 1024×1024 PNG 로 교체. iOS 기본 아이콘과 안드로이드 일부에서 베이스로 쓰입니다.

iOS 아이콘 규칙. 정사각형, 투명(알파) 금지, 모서리는 직각으로 제작(둥근 모서리는 OS가 자동 적용). 텍스트는 가장자리에서 충분히 안쪽으로.

3-2. iOS 아이콘 — .icon 번들

이 프로젝트의 ios.icon 은 Apple Icon Composer 형식(assets/expo.icon/) 으로 돼 있습니다(iOS 18/26 의 입체·다크·틴트 아이콘 지원). 두 가지 방법 중 택1:

  • 간단(권장): 그냥 PNG 하나로 쓰고 싶으면 app.json 에서 ios.icon 값을 1024×1024 PNG 경로로 변경. 
    "ios": { "icon": "./assets/images/icon.png", ... }
  • 고급: 입체/다크/틴트 변형까지 만들려면 Apple Icon Composer(무료, Xcode 16+ 포함) 로 assets/expo.icon 을 열어 편집/교체.

3-3. Android 적응형 아이콘 (전경 / 배경 / 단색)

안드로이드는 전경 + 배경을 합성해 기기마다 원형·둥근사각 등으로 마스킹합니다. 그래서 전경 로고는 가운데 안전영역 안에 둬야 잘리지 않습니다.

배경을 이미지 대신 단색으로 하려면 backgroundImage 줄을 빼고 backgroundColor 만 쓰면 됩니다.

3-4. Web 파비콘

assets/images/favicon.png 교체 (48×48, 정사각). 웹 빌드에서만 사용.

4. 스플래시 화면 (앱 실행 시 로딩 화면)

expo-splash-screen 플러그인이 처리합니다. app.json 의 플러그인 설정에서 배경색과 가운데 로고 이미지를 지정합니다.

4-1. 로고 이미지 교체

assets/images/splash-icon.png 를 투명 배경의 로고 PNG(권장 1024×1024)로 교체.

4-2. app.json 설정

[
  "expo-splash-screen",
  {
    "backgroundColor": "#208AEF",   // 스플래시 배경색
    "image": "./assets/images/splash-icon.png",  // 공통 로고 (추가 가능)
    "imageWidth": 200,            // 로고 표시 너비(dp). 현재 76 → 보통 160~220
    "resizeMode": "contain",
    "dark": {                     // (선택) 다크 모드 전용
      "backgroundColor": "#0b1f3a",
      "image": "./assets/images/splash-icon.png"
    }
  }
]
현재는 android.image 만 지정돼 있습니다. iOS·안드로이드 공통으로 하려면 위처럼 최상위 image/imageWidth 를 넣으세요. 로고가 너무 작게 보이면 imageWidth 값을 키우면 됩니다(예: 76 → 200).

5. 교체 후 반영하기

이미지를 바꾸거나 app.json 을 수정한 뒤에는 네이티브 에셋 재생성이 필요합니다.

EAS 클라우드 빌드를 쓰는 경우 (권장) — 그냥 다시 빌드하면 자동 반영
eas build --profile preview --platform android
로컬에서 네이티브 폴더를 쓰는 경우 — prebuild 재생성
npx expo prebuild --clean      # android/ ios/ 네이티브 아이콘·스플래시 재생성
잘 안 바뀌면 캐시 정리 후 재실행
npx expo start --clear
아이콘이 안 바뀌어 보이면 대개 기기/시뮬레이터 캐시 때문입니다. 앱을 완전히 삭제 후 재설치하면 새 아이콘이 보입니다. (Expo Go 에서는 절대 안 바뀜 — 실제 빌드로 확인)
